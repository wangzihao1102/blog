## 项目介绍

本项目是一个轻量级个人博客系统，采用 Flask 框架搭建。读者可以在前台浏览文章，管理员可以通过后台创建、编辑和管理博客内容。核心亮点：

- **Markdown 编辑器** — 管理后台内置 Markdown 编辑器和实时预览功能，写作体验流畅
- **标签与分类** — 支持文章标签、按标签筛选、按年月归档浏览
- **全文搜索** — 可通过标题或正文内容搜索文章
- **自动渲染缓存** — Markdown 正文保存时自动生成 HTML 并缓存，读取时无需重复转换
- **响应式设计** — 界面简洁美观，适配多种屏幕尺寸

## 技术栈

### 后端

| 技术 | 版本 | 用途 |
|------|------|------|
| **Flask** | 3.0.3 | Web 框架 |
| **Flask-SQLAlchemy** | 3.1.1 | ORM 数据库操作 |
| **Flask-Login** | 0.6.3 | 用户认证与会话管理 |
| **Flask-WTF / WTForms** | 1.2.1 / 3.1.2 | 表单验证 |
| **SQLite** | — | 开发环境数据库（实例文件夹 `instance/blog.db`） |
| **Werkzeug** | 3.0.3 | 密码哈希、WSGI 工具 |

### 前端 & 工具

| 技术 | 版本 | 用途 |
|------|------|------|
| **Python Markdown** | 3.5.2 | Markdown 转 HTML（启用 extra, codehilite, toc, attr_list, def_list 扩展） |
| **Unidecode** | 1.3.8 | 标题生成 URL 友好的 slug（支持中文转拼音） |

- **Jinja2** — Flask 自带的模板引擎，用于渲染所有 HTML 页面

## 快速开始

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 启动应用
python run.py

# 3. 访问博客
# 前台首页: http://localhost:5000/
# 管理后台: http://localhost:5000/admin/login
```

应用默认监听 `0.0.0.0:5000`，`debug=True` 开启热重载。

### 数据模型关系

```
User (1) ────< (N) Post >──── (N) Tag (M)
                      ──── post_tags (中间表)
```

- **User**: 管理员账户，存储加密后的密码
- **Post**: 博客文章，包含标题、slug、摘要、Markdown 正文、HTML 预渲染、发布状态、作者
- **Tag**: 文章标签
- **post_tags**: Post 与 Tag 的多对多关联表

## 公开端功能

公开端面向所有访客，无需登录即可浏览。

### 1. 首页 (`/`)

- 展示所有已发布文章的列表，按发布时间倒序排列
- 每篇文章卡片显示标题、发布日期、作者、摘要、标签
- 支持文章分页浏览（默认每页 10 篇）
- 顶部 Hero 区域配有搜索框，可快速搜索文章

### 2. 文章详情页 (`/post/<slug>`)

- 以渲染后的 HTML 展示文章完整内容，支持代码高亮
- 文章信息头：标题、发布日期、作者名、标签
- 底部导航：上一篇 / 下一篇快速跳转
- 返回首页入口

### 3. 标签筛选 (`/tag/<slug>`)

- 点击任意标签可进入标签页面
- 展示该标签下所有已发布文章
- 同样支持分页

### 4. 搜索功能 (`/search?q=关键字`)

- 在首页搜索框输入关键字，按回车搜索
- 对文章的标题和正文进行全文模糊匹配
- 展示匹配的文章列表，无结果时提示"没有找到相关文章"

### 5. 文章归档 (`/categories`)

- 按年份分组展示归档列表
- 点击年份展开月份链接（1-12 月）
- 按月浏览：`/archive/<year>/<month>`，如 `/archive/2026/7`
- 月份标题使用中文（一月、二月…十二月）

### 导航栏

顶部导航栏常驻显示，包含：
- **首页** — 回到文章列表
- **归档** — 进入分类归档页
- **搜索** — 聚焦到首页搜索框
- **登录 / 管理** — 未登录显示"登录"，登录后显示"管理 + 退出"

## 管理端功能

管理端位于 `/admin` 路径下，访问前必须通过登录认证。

### 登录 (`/admin/login`)

- 用户名 + 密码表单登录
- 支持"记住我"选项（持久化会话）
- 登录失败时显示中文错误提示
- 若用户已登录则自动跳转到仪表盘

### 仪表盘 (`/admin/dashboard`)

登录后的管理中心，提供以下功能：

#### 统计卡片

| 指标 | 说明 |
|------|------|
| 总文章数 | 所有文章数量（含草稿） |
| 已发布 | 已公开发布的文章数 |
| 草稿 | 未发布的草稿数量 |

#### 最近文章列表

- 展示最近的 5 篇文章
- 每行显示：标题、发布状态徽章（已发布/草稿）、创建日期
- 操作列提供三个按钮：
  - **编辑** — 进入文章编辑页
  - **查看** — 在新标签页打开公开版文章详情页
  - **删除** — 弹出确认对话框后删除文章

#### 快捷操作

- **新建文章** — 跳转到文章编辑页
- **管理标签** — 跳转到标签管理页

### 新建 / 编辑文章 (`/admin/posts/new` / `/admin/posts/<id>/edit`)

文章编辑器是全功能 Markdown 写作工具：

#### 编辑表单字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 标题 | 文本 | 是 | 最多 200 字符 |
| 摘要 | 文本 | 否 | 最多 300 字符，用于文章列表展示 |
| 正文 | 多行文本 | 是 | 使用 Markdown 格式编写 |
| 标签 | 文本 | 否 | 多个标签用逗号分隔（如 `python, flask, 教程`） |
| 立即发布 | 复选框 | — | 勾选则直接发布，不勾选则为草稿 |

#### 编辑器特性

- **Markdown 实时预览** — 点击"切换预览"按钮，下方显示渲染后的 HTML 效果
  - 同时支持输入时自动刷新预览（防抖 500ms）
  - 预览通过 AJAX 异步请求后端渲染，无需页面跳转
- **自动 slug 生成** — 根据标题自动生成 URL 友好的 slug，中文标题会自动转为拼音
- **slug 唯一性处理** — 标题冲突时自动追加 `-1`, `-2` 等后缀
- **自动保存 body_html** — 保存文章时，Markdown 正文自动转换为 HTML 并缓存
- **智能标签管理** — 提交时自动解析逗号分隔的标签字符串，已存在的标签复用，不存在的自动创建
- **标签输入提示** — 前端提供已有标签的下拉建议

### 标签管理 (`/admin/tags`)

一站式标签管理页面：

- **添加标签** — 输入标签名称，点击"添加"即创建
  - 自动检查重复（按 slug 判断），重复时给出提示
  - 空名称会被拦截
- **标签列表** — 表格展示所有标签的名称和 slug
- **删除标签** — 每行有删除按钮，确认后移除标签

### 登出 (`/admin/logout`)

- 清除登录会话
- 重定向到博客前台首页

### 安全机制

- 所有 `/admin/*` 路由通过 `@login_required` 装饰器保护
- 页面导航栏根据登录状态动态切换（登录 → 显示"管理+退出"，未登录 → 显示"登录"）
- 密码使用 Werkzeug 的 `generate_password_hash` / `check_password_hash` 做哈希存储，**明文不存库**
- Session 保护级别为 `strong`（防止 Session 劫持）

## 环境变量配置

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `SECRET_KEY` | `dev-secret-key-change-in-production` | Flask 密钥，生产环境务必修改 |
| `ADMIN_USER` | `admin` | 管理员默认用户名 |
| `ADMIN_PASS` | `changeme` | 管理员默认密码 |
| `DATABASE_URL` | `sqlite:///instance/blog.db` | 数据库连接串，可改为 PostgreSQL / MySQL 等 |

设置方式示例：

```bash
export SECRET_KEY='your-secure-random-key'
export ADMIN_USER='myblog'
export ADMIN_PASS='super-secret-password'
python run.py
```

> **生产部署提示：** 请务必修改 `SECRET_KEY`、管理员默认凭据，并将 `DATABASE_URL` 切换为生产数据库。不要使用 `debug=True`。
